Motion
The signature motion is the hero glyph itself: ~42,000 Three.js Points sampled from the letter's own rasterized outline, each running a critically-damped spring toward its target (damping 0.68 at rest, 0.80 mid-morph) — so a letter dissolves and reforms in a staggered wave (per-particle random seed delays start up to 0.42 of the timeline) rather than a hard cut. Cursor is a true 3D repeller: the pointer is unprojected via raycaster onto the z=0 glyph plane, particles inside a radius are pushed radially AND lifted off-plane in z to carve a hole with real depth, and the same formation spring heals it the instant the cursor leaves. Click fires a one-shot decaying ripple that shoves grains outward as the next glyph casts. Easing declared as CSS custom props: --ease cubic-bezier(.22,.61,.36,1) and --ease-io cubic-bezier(.7,0,.3,1) — smooth, weighted, non-bouncy. Motion is meaning here: the type literally behaves like poured/settling matter, matching the "Type is matter" thesis.
Advanced technique
WebGL2 (WebGL2RenderingContext) driven by Three.js 0.160 (three.module.js, __THREE__ global) rendering a single full-viewport canvas (1429x900) as a THREE.Points particle system — 42k particles desktop / 24k mobile. No font-parsing library: glyphs are rasterized to an offscreen 2D canvas at Fraunces 900, pixels alpha-tested (alpha>128) on a 2px stride with per-cell jitter into world-space particle targets; fixed particle budget mapped onto each glyph's sample count via modulo (i % n) so every letter uses every particle. Waits on document.fonts.ready then re-samples to avoid sampling the fallback face. CSS uses mix-blend-mode and filter (14.9KB stylesheet); ~7% of particles tinted vermilion as accent. Ships a static-letter fallback if WebGL context is unavailable and reports a clean headless render. data-engine / data-g attributes drive the glyph swap. Minimal footprint: two scripts (app.js + three.module.js), 104 bytes inline.

Guide-page DNA (own words)

The /guide/ page is a build log framed as a foundry process note. Premise: the specimen is not a picture of type, it is the type, rebuilt every frame from ~40k points of ink. It documents a reproducible technique in six numbered steps. (01) Brief: a type foundry whose specimen IS the particle system — a live letter cloud that samples its own outline, settles, reflows to the next glyph, and yields under the cursor; the governing constraint is that the letter must read crisply AS a letter from a single static screenshot frame, which forced dense sampling, high damping, and clean counters. (02) Sampling: skip font-parsing libraries entirely — render the glyph heavy to an offscreen 2D canvas, walk pixels, emit a particle wherever alpha is opaque; stride controls density, jitter kills the grid look, and you must wait for fonts.ready or you sample the wrong typeface. (03) Morph: per-particle random seeds stagger the start so the form dissolves in a wave; each point is a critically-damped spring, damped hard at rest for a crisp screenshot. (04) Interaction: unproject the pointer onto the glyph plane with a raycaster for a genuine 3D repeller that carves depth in z, letting the formation spring do the healing for free. (05) Process: three screenshot-driven passes, critiqued "like a hostile art director" — read-but-drifting → tightened resting spring; comet trails + mobile collision → damping 0.68, mobile canvas shrinks to 60% with a paper scrim and collapsed controls, added click ripple; final polish tuned vermilion to ~7% and verified zero console errors + headless WebGL with static fallback. (06) A seven-point reproduce-this checklist. The underlying DNA: concept-driven (the interaction literally embodies the tagline), print-craft vocabulary, and a screenshot-judged quality bar where the resting frame must survive a still critique.

Glyph deep-dive (the favorite)

Glyph does six things the other nine do not, and together they explain why it tops the editor's list.

1) The specimen IS the product, with a rigor the others' procedural systems don't reach. Others generate content (daedala's PRNG maze, escalier's stairs, mordant's dye field), but glyph maps a FIXED particle budget (42k desktop / 24k mobile) onto every glyph via modulo (i % n) so every letter reuses every particle, and it skips font-parsing entirely — it rasterizes Fraunces 900 to an offscreen 2D canvas, alpha-tests pixels (alpha>128) on a 2px stride with jitter, and waits on document.fonts.ready so it never samples the fallback face. The hero letter (confirmed in the screenshot as a dissolving 'A' with corner labels NOW CASTING / PARTICLES 42,000 / STATE SETTLING / MARROW · 144PT · BLACK) is not a picture of type — it is the type, rebuilt every frame from ink.

2) It is the only site with a genuine 3D-depth cursor interaction. The pointer is unprojected via raycaster onto the z=0 glyph plane and particles lift OFF-plane in z to carve a hole with real depth, then the same formation spring heals it when the cursor leaves. shaketable (skewX transforms), mordant (2D dye), unsaid (2D scatter) are all planar; escalier has 3D but it's a camera shear, not per-element depth carving. Interaction that carves and heals is more physically committed than anything else in the set.

3) It fuses two things no other site fuses: a live particle demo AND a real, readable type-specimen sheet. Screenshot 2 shows an actual weight ladder rendered in Fraunces itself — LIGHT·300 'Handset', REGULAR·400 'Foundry', MEDIUM·500 'Matrix', BOLD·700 'Casting' — plus a bordered glyph-cell case (A G R & @, visible bottom-right of the hero) and an 'AT TEXT SIZE' running block. It is simultaneously a working foundry catalogue and a WebGL toy; the others commit to one register.

4) Restraint is quantified to a number. Only ~7% of particles are tinted vermilion (#e3402a); the italic Fraunces cut carries the single red emphasis word ('matter.'). This is the most explicit statement of the accent-budget discipline that recurs across the set, and it reads directly in the hero.

5) Compositional RANGE within one page. Screenshot 3 shows the palette inverting to a full warm-ink #17140f 'License the family' section with three bordered pricing cards (Desktop $180 / Studio $540 with a red 'MOST POURED' pill / Broadcast 'Let's talk'), then flipping back to ivory for the footer ('How this was built →', 'Designed & built by Claude Fable 5'). It proves light AND dark composition in one document without a second palette.

6) Crucially, the conceit is PLAUSIBLE-REAL, not absurdist. A digital-and-metal type foundry (EST. MMXXIII) is a believable business; escalier, mordant, daedala and lamella are deadpan-impossible institutions. Glyph gets the authored-intent and worldbuilding payoff without the joke, so it reads as commercially and technically credible — which is exactly what a recruiter-facing portfolio needs.

Why it likely wins: glyph is the maximal fusion the editor wants CoreWise to emulate — undeniable engineering (WebGL2 + Three.js 0.160, raycaster depth, critically-damped springs, headless-render fallback) married to the most disciplined print craft in the set (real specimen sheet, № and roman-numeral conventions, drop cap, quantified 7% accent, weight ladder), wrapped in a screenshot-survivable resting state (damping raised to 0.68 so the settled letter photographs crisply). It satisfies both audiences at once: the recruiter sees verifiable hard front-end; the editor sees a print-trained type specimen; and neither sees slop or a gimmick. It demonstrates its thesis instead of asserting it, which is the whole game.

CoreWise direction — buildable rules

  1. GROUND: warm ivory/bone paper as the default surface (target ~#f2ece1–#f4f1ea), warm near-black ink for text (~#17140f–#1a1712). Never pure #fff or #000. Reserve exactly one full-bleed dark-inverted band per page (warm charcoal, not black) for a single high-emphasis moment (enrollment CTA, key result, colophon), mirroring glyph's license section.
  2. TYPE, DISPLAY: one high-contrast variable literary serif (Fraunces / Instrument Serif register), used large with negative tracking (roughly -2px at ~100px). Its italic cut carries emphasis and the accent-colored payoff words. This is the publication's voice.
  3. TYPE, UTILITY + DATA + CODE: one mono or mono-adjacent grotesk (Space Grotesk / true mono) for eyebrows, section numbers, figure captions, dataset/eval numbers (tabular tnum figures), and all code. Never set long body copy in mono.
  4. TYPE, BODY: a comfortable reading face at 16–17px / ~1.6 line-height for running prose. Two families total (three only if a distinct 'handwritten margin note' role earns it, per lamella's Caveat).
  5. ACCENT: exactly one, given a written job description and a coverage budget (~5–10%, glyph's 7% / kiln's ~5 uses as the model). Spend it on emphasis words, key numerals/results, links, one drop cap per feature, and the single most important state in each figure. Pick an ownable hue — the reference set is saturated with warm reds/terracottas, so consider differentiating rather than copying vermilion.
  6. SIGNATURE INTERACTIVE: one bespoke WebGL/canvas piece per flagship lesson that DEMONSTRATES that lesson's concept (tokens fragmenting, attention as a draggable weighted graph, embeddings settling into a projection). The figure is the lesson's first sentence, never decoration behind body text. Everything else on the page stays quiet.
  7. 3D BUDGET: keep heavy WebGL/Three.js to that single hero figure — one canvas, with a static fallback for no-WebGL and a meaningful, legible first frame on load (seeded pre-roll, never a blank/grey rectangle). Do not run particle or physics loops behind reading content.
  8. LAYOUT: hairline broadsheet grid — separate entries and cells with 1px warm rules (drop every 3n right border, kiln-style) and whitespace. No elevated drop-shadow cards, no rounded glassmorphism. Left-anchor type on a strict baseline; hold prose measure tight (~500–660px) even inside full-bleed sections.
  9. PRINT APPARATUS: number lessons and sections visibly (01–NN, §); carry an issue/volume mark in № + roman numerals; open flagship features with a drop cap; render benchmark/eval data as real ledger tables; use the wide right margin for definitions, citations and asides.
  10. MOTION: one named easing token (expo-out, ~cubic-bezier(.22,1,.36,1)) at ~0.4s for all UI transitions — color/transform/opacity only, nothing bounces in reading zones. Scroll-reveal via IntersectionObserver, unobserve-once.
  11. ACCESSIBILITY IS NON-NEGOTIABLE: honor prefers-reduced-motion (layout and legibility fully intact when motion stands down), pause every canvas loop on document.hidden, maintain high contrast on warm paper.
  12. TEXTURE: procedural only — SVG feTurbulence paper grain at ~3% multiply, CSS-drawn accents; diagrams as inline SVG. No stock textures or decorative stock imagery.
  13. ONE SOURCE OF TRUTH: any interactive figure and its accompanying static diagram/prose must be driven by the same data/model so a figure can never contradict what the lesson says (lamella/daedala/bindery principle). Critical for teaching accuracy.
  14. VOICE: authored, precise, plausible-real. Teach real AI earnestly; do NOT wrap the publication in an absurdist fictional-institution conceit. Let craft, restraint and a demonstrated concept — not a joke — be the differentiator (glyph's plausible foundry, not escalier's impossible staircases).
  15. SHIP THE COLOPHON: for flagship lessons, publish a 'how this was built / how we teach this' methodology note documenting decisions, the figure's technique, sources and cuts — process transparency as recruiter-facing proof, mirroring the universal /guide build-log.

Open questions for the editor

  • Light warm-paper ground or dark matte-charcoal? The editor's admired set splits — glyph/kiln/unsaid/escalier/daedala are light warm paper, while lamella (#16181c) and mordant (#26231e) are dark. The stated favorite is light, which suggests a default, but does CoreWise want a dark mode or dark flagship pieces as an alternate register?
  • Display voice: literary high-contrast serif (glyph Fraunces, kiln Instrument Serif, unsaid Fraunces), inscribed Roman capitals (escalier/daedala Cinzel), or industrial condensed (shaketable Big Shoulders)? These signal very different personalities — refined-editorial vs monumental-institutional vs industrial. The favorite points to literary serif, but this should be confirmed as the brand voice.
  • Accent hue: nearly every favorite converges on a warm red/vermilion/terracotta/oxblood. Does CoreWise deliberately own that same red family (accepting kinship with the references) or pick a distinct, ownable accent so it doesn't read as derivative of the studied set?
  • How much whimsy/conceit is on-brand for a real learning publication? The favorite (glyph) uses a plausible-real conceit (a type foundry); the others are absurdist institutions. Where exactly is the editor's line between 'authored personality' and 'joke that undercuts a real curriculum'?
  • Engineering-heavy WebGL signature (glyph/escalier/mordant/lamella) vs zero-library CSS-craft restraint (kiln/unsaid/rawmatter)? Both are admired. For an audience of AI recruiters judging an AI publication, does a live shader/3D hero prove more, or does framework-free restraint read as more senior? This drives the whole tech budget.

Does the editor want one consistent template applied across all lessons, fully bespoke heroes per flagship piece, or a two-tier model (bespoke flagships + a restrained system for the long tail)?
  • Does the reading experience tolerate ANY ambient motion behind long-form text, or must all motion be quarantined to isolated hero figures? The references keep motion in heroes/figures, but none carries the volume of sustained instructional prose CoreWise will — the editor's tolerance here should be set explicitly.
